Philosophy

The One-Engine Law

Write the rule once in pure F#. Run it natively on backend servers for massive throughput. Compile it directly into WebAssembly to validate offline in the browser. Zero drift.

⚡

Zero Latency

By bypassing heavy generic databases and relying on pure compiled F# discriminated unions, GSTFlow validates enormous payloads in sub-milliseconds. No network. No round-trips.

📦

The Universal Envelope

Every complex Indian tax rule resolves into the mathematically strict VerdictEnvelope. Pass, Fail, Warning, Unknown — no ambiguity. Just pure, auditable state.

🛡️

Absolute Provenance

GSTFlow doesn't just reject an invoice. It cites the exact paragraph and section of the CGST Act, the effective date range, and the mathematical evidence governing the violation.

🌐

Portable by Design

The same F# rule that runs on a .NET 10 server compiles to JavaScript via Fable and runs in WebAssembly in your browser. Identical semantics. Zero server dependency.

🔬

Battle-Tested

Snapshot golden tests verify exact semantic correctness. FsCheck property-based fuzzing throws thousands of random mutations at the engine. It has never crashed.

📖

Fully Open Source

MIT licensed. No vendor lock-in. No proprietary formats. Every rule, every evidence trail, every architectural decision is transparent and auditable by the community.

Quick Start

See It In Action

GSTFlow.Cli — validate invoice.json 
// 1. Define your invoice
let invoice = Invoice.fromJson invoicePayload

// 2. Run the deterministic engine
let envelope = GSTFlow.Rules.evaluate invoice

// 3. Get the mathematical verdict
match envelope.OverallOutcome with
| Pass    → "✅ Guaranteed Compliance."
| Fail    → "❌ Instant Mathematical Rejection."
| Warning → "⚠️ Compliant with caveats."

// Every result carries full provenance:
//   Rule ID, CGST Act Section, Effective Date,
//   Expected vs. Observed values, Confidence level
